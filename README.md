## 📝 وصف المشروع (Project Summary)
يهدف النظام إلى أتمتة وتبسيط كافة العمليات المتعلقة بإدارة رحلات الطيران وحجوزات المسافرين. يدمج النظام بين واجهات مستخدم تفاعلية وقاعدة بيانات متينة لإدارة الرحلات والمدفوعات الحية، معتمداً على مبادئ البرمجة كائنية التوجه لضمان مرونة الكود وسهولة صيانته وتحديثه.

### 👥 أطراف النظام (Actors):
- **المسافر (Customer/Passenger):** يمكنه البحث عن الرحلات المتاحة، عرض تفاصيل الأسعار والمواعيد، وإنشاء حساب شخصي، وإتمام الحجوزات والمدفوعات المالية.
- **مسؤول النظام (Admin):** يملك صلاحيات كاملة لإدارة أسطول الطائرات، إضافة وتعديل وحذف الرحلات الجوية، ومراقبة جدول الحجوزات والعمليات المالية المدمجة.

## 🛠️ المفاهيم البرمجية المطبقة (OOP Concepts Implemented)
تم بناء هيكلية المشروع لتغطية المتطلبات البرمجية الأساسية للمادة بنجاح:
1. **Encapsulation (الكبسلة):** عُرفت جميع خصائص الكائنات كـ `private` مع توفير دوال الوصول والتعيين `getters/setters`.
2. **Inheritance (الوراثة):** استخدام كلاس أب مجرد `User` يرث منه كلاس الـ `Admin` وكلاس الـ `Customer` لتجنب تكرار الكود.
3. **Polymorphism (تعدد الأشكال):** تم عبر دمج الكلاسات المجردة (Abstract Classes) وإعادة تعريف الدوال (Method Overriding) لتقديم سلوكيات ديناميكية لكل مستخدم.
4. **Exception Handling:** تضمين كتل `try-catch` الذكية لمعالجة أخطاء إدخال البيانات وأخطاء الاتصال بقاعدة البيانات لضمان استقرار تشغيل النظام.
5. **Database Connection:** ربط كود الجافا بالكامل بقاعدة البيانات عبر تقنيات JDBC ومزامنة البيانات حياً.
6. **Javadoc Documentation:** تم توليد توثيق برمي شامل لجميع الكلاسات والدوال بصيغة HTML.
